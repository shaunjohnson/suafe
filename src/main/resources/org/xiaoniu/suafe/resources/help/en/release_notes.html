Release 1.1.0 - 2007.08.12

What's new in this release:

  • Suafe now has a command line interface. All edit and report functions are available through a scriptable, command line interface. Input/output directly from/to files or through STDIN and STDOUT.
  • A recent files list feature has been added. This lists out the last 10 files that were edited using Suafe.
  • The last file edited is opened automatically upon startup. This feature can be disabled.
  • New reports:
    • Summary Report provides a comprehensive, HTML summary of all the information within an authz file.
    • Statistics Report provides a simple summary of user, group, repository and access rule counts.
  • Upon saving changes Suafe warns you if any unreferenced users, repositories or paths exist.
  • Fonts are now configurable. You're no longer limited to the default Java font. By default all values are displayed in a monospace font.
  • User, Group and Access Rule counts are displayed in the title of each tab. A quick reference to see how many of each object exists.
  • Clone user and clone group buttons are now available on the users and groups tabs.
  • Preview and report screens are now resizeable.
  • Tables are now sortable by column. Just click the header within a table to resort the data.
  • Table column widths now size themselves to more accurrately display the data within the table.
  • Application status bar at the bottom of the main window displays the full path of the file currently being edited.
  • Window size, location and state as well as divider location are now saved when the application is closed. These saved settings can be cleared at anytime.
  • The application now checks for circular references. i.e. If group A is a member of group B then group B cannot be a member of group A or any group that is a member of group A.
  • Change Membership and Add/Remove Users dialogs are now resizeable. Resize the dialog to accommodate larger amounts of data.
  • Drag and drop an authz file from the operating system onto Suafe to open it.
  • You may now click the delete key to delete items and change group memberships.

What's fixed in this release:

  • Dialogs such as the add/edit access rule dialogs now resize themselves according to the size of the data.
  • Line numbers reported within error messages displayed when parsing/opening a file now correctly indicate the line number where the problem was encountered.
  • Bug where group definitions contained other groups were not being parsed correctly has been addressed.
  • Bug where a user and groups appear repeatedly if group or user is assigned to the same group more than once.

Release 1.0.0 - 2007.01.27

What's new in this release: 

  • Bug: Allow deletion of server level access rules
  • Suafe starts maximized
  • All dialogs and the help screen can be closed with a hit of the escape key
  • Suafe now keeps track of changes that you make. It'll prompt you to save the file before doing anything that would clear the memory like closing the application
  • More help screens
  • Fixed bug with edit access rule
  • Fixed bug where the screens didn't change when using cursor keys
  • Widened the clone dialogs. Added an icon for the clone menu items
  • Corrected a problem with the resource files

What's gone missing in this release:

  • Repository browser has been removed. This wasn't available so nobody should miss it. Trouble getting the browser to work properly and poor performance of the JavaSVN library lead me to remove this feature. It is also something that isn't all that important; I don't think it would be used much.
  • The print function was removed. Printing an authz file is interesting, but again not important. It's more important to have a good editor than to be able to print files which are unlikely to be read by anybody.
